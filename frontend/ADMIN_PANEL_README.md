## ğŸ” Authentication Flow

1. Admin navigates to `/admin/login`
2. Enters credentials (default: `admin@zenith2026.com` / `Admin@2026`)
3. On successful login:
   - JWT token stored in localStorage
   - Redirects to `/admin/dashboard`
4. Protected routes check for valid token
5. Logout clears token and redirects to login

## ğŸ“¤ Upload Workflow

1. Admin selects file (image or video)
2. Fills in metadata:
   - Title (required)
   - Description (optional)
   - Category (event/sports/ceremony/participants/other)
   - Tags (comma-separated)
3. Clicks "Upload Media"
4. Frontend sends FormData to backend
5. Backend uploads to Cloudinary
6. Cloudinary URL stored in MongoDB
7. Success message displayed

## ğŸ“ Usage Guide

## ğŸ› Troubleshooting

### "Unauthorized" error

- Check if JWT token exists in localStorage
- Try logging in again
- Verify backend is running

### Media not uploading

- Check file size limits
- Verify Cloudinary credentials in backend `.env`
- Check browser console for errors
- Ensure backend is running

### Images not displaying

- Verify Cloudinary URLs are valid
- Check CORS settings in backend
- Inspect network tab for 404s
